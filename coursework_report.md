## Цель работы
Создать персональный сайт-визитку с помощью MkDocs и темы Material, настроить навигацию, наполнить контентом и опубликовать на GitHub Pages.

## Выполненные шаги

1. **Установка и настройка окружения**
   - Установлен Python
   - Установлены MkDocs и тема Material: `pip install mkdocs mkdocs-material`
   - Создан новый проект: `mkdocs new .`

2. **Настройка конфигурации (`mkdocs.yml`)**
   - Указаны название сайта, описание, автор
   - Подключена тема Material с настройками (навигационные вкладки, поиск, переключение тем)
   - Настроено меню из четырёх разделов: Главная, О себе, Проекты, Контакты
   - Добавлены иконки соцсетей и информация об авторских правах

3. **Создание контента**
   - **Главная (`index.md`)** – приветствие, краткая информация, список навыков, цитата, фото
   - **О себе (`about.md`)** – образование, увлечения, личные качества (с таблицей)
   - **Проекты (`projects.md`)** – описания трёх проектов с технологиями и ссылками на репозитории
   - **Контакты (`contacts.md`)** – email, Telegram, GitHub

4. **Добавление изображений**
   - Создана папка `docs/images`
   - Добавлено личное фото (отображается на главной странице)
   - Использован синтаксис Markdown для вставки и изменения размера изображения

5. **Локальное тестирование**
   - Запущен сервер разработки: `mkdocs serve`
   - Проверены все страницы, ссылки, адаптивность

6. **Публикация на GitHub Pages**
   - Репозиторий инициализирован и связан с удалённым
   - Сайт собран в папку `public`: `mkdocs build --site-dir public`
   - Создана ветка `gh-pages` и в неё скопированы файлы сайта
   - Ветка отправлена на GitHub с принудительным пушем
   - В настройках репозитория включена GitHub Pages для ветки `gh-pages`
